# AI 25 – ECG Data Analysis  

## Тема проекта  
**Анализ данных ЭКГ**  

## Цель проекта  
Создать сервис для анализа данных ЭКГ с использованием методов машинного и глубокого обучения, способный выявлять аритмии и прогнозировать риски сердечно-сосудистых событий.

- Ростов Павел

## План работы  

### Этап 1. Установочный  
- Создать чат с куратором  
- Определиться с предметом исследования  
- Создать репозиторий и оформить документацию  
- Подготовить годовой план  

### Этап 2. Разведочный анализ данных (EDA)  
**Base:**  
- Сбор открытых датасетов  
- Базовый EDA: статистики, визуализация распределений  
- Обработка пропусков и аномалий  

**Middle:**  
- Спектрограммы, матрицы корреляций  
- Первичный feature engineering  

### Этап 3. Линейные ML-модели  
**Base:**  
- Logistic Regression (норма/патология)  
- Linear SVM (обнаружение аритмий)  
- Спецификация метрик: приоритет минимизации **ложноотрицательных** результатов 

**Middle:**  
- Ridge/Lasso Regression (вероятность аритмий)  
- SVM с линейным ядром (3–5 классов аритмий)  

### Этап 4. Нелинейные ML-модели  
**Base:**  
- Random Forest (5–7 классов аритмий)  
- Gradient Boosting (риск сердечных событий)  
- XGBoost (предсказание фибрилляции предсердий)  
- Сравнение с линейными моделями  

**Middle:**  
- Углубленный feature engineering  
- Отбор наиболее значимых признаков  

### Этап 5. Сервис  
**Base:**  
- Разработка прототипа Telegram-бота для загрузки и анализа ЭКГ  

### Этап 6. Глубокое обучение (DL)  
**Base:**  
- Классификация по сырым сигналам ЭКГ  
- Выявление сложных паттернов аритмий  

**Middle:**  
- Прогнозирование эпизодов аритмии  
- Моделирование последовательностей аритмий  

### Этап 7. Улучшение моделей и сервиса  
**Base:**  
- Отработка замечаний экспертов  
- Финальная тренировка лучшей модели  
- Тестирование и улучшение сервиса  

**Middle:**  
- Контейнеризация (Docker)
